# Obsidian Live Syncinator Server

A real-time collaboration server for Obsidian vaults. Handles file synchronization via WebSocket with Operational Transformation (OT), file storage on disk, and workspace-based multi-tenancy backed by SQLite.

## Development Workflow

We practice Extreme Programming. TDD is not optional.

### Red-Green-Refactor

1. **Red**: Write a failing test that describes the desired behavior
2. **Green**: Write the minimal code to make it pass
3. **Refactor**: Clean up while keeping tests green

**Tests must test BEHAVIOR, not implementation.** Ask: "what should happen when the user does X?" not "does function Y call function Z?"

### Commits

One fix per commit. Commit message format follows the existing convention:

```
fix: short description of the change
feat: short description of the new feature
refactor: short description of the refactoring
```

### Code Style

- No unnecessary comments. Code should be self-explanatory.
- Only add comments when explaining **why**, never **what**.
- No over-engineering. Solve the problem at hand, nothing more.
- Run `make lint` and `make test` before committing.

## Architecture

- `cmd/main.go` — HTTP server entrypoint
- `cmd/cli/main.go` — CLI tool for workspace management
- `pkg/` — Core logic: API handlers, WebSocket, subscriber, diff/OT
- `internal/repository/` — SQLite queries (generated by sqlc)
- `internal/migration/` — Database migrations (goose)
- `pkg/filestorage/` — Disk-based file storage with path traversal protection

## Running

```bash
make test    # Run all tests
make lint    # Run golangci-lint + SQL formatter
```

## SQLite Configuration

The server enables `PRAGMA journal_mode=WAL` and `PRAGMA foreign_keys = ON` at startup. Tests seed workspaces with IDs 1, 2, 10, 11, 123.
